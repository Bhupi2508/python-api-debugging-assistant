# python-api-debugging-assistant

## Project Overview

`python-api-debugging-assistant` is a **console-based (CLI) Python app** that helps developers:

- capture API error details (URL, method, status code, request/response bodies, etc.)
- inspect the captured information
- later analyze it with AI (introduced in a future phase)

## Objectives

1. Build a clean, beginner-friendly CLI workflow.
2. Collect consistent input from users (developers).
3. Prepare the project to grow step-by-step into a bigger system (database, export, AI, FastAPI).

## Current Phase

**Phase 1: Console Input + Documentation**

- You will learn how the CLI app should behave.
- We will describe menus, prompts, validation ideas, and the conceptual storage/analysis steps.
- No database, no AI, and no API endpoints are implemented in Phase 1.

## Future Phases

- Phase 2: Functions
- Phase 3: Modules
- Phase 4: SQLite
- Phase 5: Logging
- Phase 6: Import / Export
- Phase 7: AI Integration
- Phase 8: FastAPI
- Phase 9: Authentication
- Phase 10: Deployment

## Learning Goals

By the end of Phase 1, you should understand:

- how a CLI program interacts with a user
- how to validate user input (at a conceptual level)
- how to think about incremental project growth
- how the “same workflow” can later be reused in a web app
